# 采集运行链路

状态：已完成

## 目标

实现从阶段计划到批次关闭的采集闭环，所有 Tushare 调用统一经过 Provider、全局额度和持久化任务状态。

## 实现任务

1. 实现交易日门禁、MASTER/DAILY/HOT/DELAYED/BACKFILL/REPAIR 阶段计划器。
2. 在同一事务中幂等创建批次任务并冻结计划版本和预期任务数。
3. 使用短事务条件更新领取 PENDING 或到期 RETRY_WAIT 任务。
4. 在事务外执行 Tushare 请求、分页拆分、完整性判断和 Parquet 封存。
5. 实现物理请求重试与任务级延迟重试，分类不可恢复错误。
6. 实现批次关闭、部分失败语义、启动协调和容量反压。
7. 使用已配置 Token 完成受控接口连通性和沙盒采集验证。

## 验收条件

- 请求频率平滑控制在 480 次/分钟预算内。
- HTTP 成功但数据不完整时任务不能成功。
- 批次未冻结或任务数不符时不能关闭。
- 原批次关闭后不重开，迟到数据进入 REPAIR 批次。
- Scheduler 重启能够恢复 RUNNING 任务和资产中间状态。
