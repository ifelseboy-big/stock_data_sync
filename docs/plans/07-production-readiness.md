# 生产验收与恢复

状态：实现中

## 目标

完成 Mac mini 上线前的安全、容量、性能、备份和故障恢复验证，使安装脚本之外的运行保障真正闭环。

## 实现任务

1. 实现 PostgreSQL 逻辑备份、Parquet 增量备份、清单和 SHA-256 校验。
2. 实现磁盘预警、保护阈值和采集反压。
3. 执行 Scheduler、采集、加工、数据库不可用和磁盘写满恢复演练。
4. 使用目标数据量验证 COPY、分区、索引、队列领取和管理查询性能。
5. 检查数据库角色、文件权限、Token 脱敏、防火墙和管理接口访问控制。
6. 在干净 Mac mini 上执行发布包校验、首次安装、重启恢复和版本升级演练。

## 验收条件

- 备份不只存在生产磁盘，每月恢复演练通过。
- 达到保护阈值后历史任务和非紧急采集按规则暂停。
- 进程或主机重启不会产生重复发布或丢失已封存资产。
- `start`、`stop`、`restart`、`status`、`logs`、`doctor` 和 `migrate` 全部通过。
- RPO 不超过 24 小时，RTO 不超过 4 小时。

## 当前实现进度

- 已实现外部目标强制校验、PostgreSQL custom-format 逻辑备份、Parquet SHA-256 增量/全量备份、完整清单与逐文件校验。
- 已实现恢复到临时数据库后切换名称、跨安装目录原始资产 URI 改写、原始目录原子切换，并保留恢复前数据库和目录。
- 已在临时 PostgreSQL 上完成真实 `pg_dump`、`pg_restore`、数据库切换和 Alembic 一致性恢复演练；分区库不能直接 `pg_restore --clean` 的问题已通过临时库方案规避。
- 容量预警、保护阈值和采集反压已在采集阶段实现并有单元测试。
- `doctor` 已增加配置权限、Token、请求预算、服务用户目录权限、PostgreSQL 和迁移版本检查。
- 安装器支持用户指定外部 `--backup-dir`，并生成每日 03:00 的 launchd 短任务；`doctor` 同时检查备份目录和服务用户写权限。
- 已实现可重复的隔离 PostgreSQL 性能脚本；覆盖千万级宽字段股票日线、50 万采集任务、10 万加工任务、30 万依赖、100 万物理请求记录、真实 COPY 发布、分区裁剪、队列领取和全部管理查询。
- 目标规模在当前 arm64 macOS 开发机 PostgreSQL 18.4 上已通过，详细规模、阈值和结果见 [目标规模性能验收](07-performance-results.md)。
- 开发、性能测试和生产部署已统一为当前最新稳定主版本 PostgreSQL 18。干净 Mac mini 上的首次安装、重启和升级演练仍未完成，因此本阶段保持“实现中”。
