# 实现规划

本目录用于把已经确认的系统设计拆成可执行、可验收的实现计划。架构规则仍以 `docs/` 顶层设计文档为准，计划文件只描述实施顺序和交付范围，不在这里维护第二套设计。

## 文件命名

```text
NN-topic.md
```

- `NN`：两位顺序号，表示建议实施顺序。
- `topic`：使用简短英文名称，例如 `01-runtime-schema.md`。
- 一个文件只规划一个能够独立验收的实现主题。

## 计划内容

每份计划至少包含：

1. 目标与非目标。
2. 前置依赖和涉及模块。
3. 实现任务及先后关系。
4. 数据库迁移、配置或部署影响。
5. 测试与验收条件。
6. 风险、回滚方式和遗留事项。

## 状态

计划状态统一使用：`待确认`、`已确认`、`实现中`、`已完成`、`已暂停`。

设计尚未确认时只能编写计划草稿，不能进入实现；完成后必须记录对应提交和验收结果。
