# 运维后台闭环

状态：已完成

## 目标

用真实运行表和指标替换管理端占位数据，形成查询、定位、人工处置和审计闭环。

## 实现任务

1. 实现概览、采集批次、加工队列、依赖、发布、接口监控、运行记录和告警查询。
2. 所有列表增加时间范围、过滤、分页和查询索引保护。
3. 实现补采、回填、重试、跳过和取消命令，要求原因与幂等键。
4. 增加管理 Token 鉴权、请求审计和敏感字段脱敏。
5. 对齐前后端契约，删除页面占位数据并补齐加载、空状态和错误状态测试。
6. 实现数据库、调度器、目录容量和服务版本状态页面。

## 验收条件

- 页面展示指标与文档口径一致，零分母返回 `null`。
- 浏览器不直接调用 Tushare、不更新正式表、不操作原始文件。
- 重复人工请求返回同一资源，不产生重复批次或任务。
- 每次写操作能够追溯操作者、原因、请求 ID 和结果。

## 实现结果

- 已实现运维聚合、发布、物理接口请求指标和系统资源查询，列表采用 30 天受限范围、服务端过滤与分页。
- 已实现历史回填、修复批次、采集/加工重试、跳过、取消和批次取消；管理命令只写 PostgreSQL 队列。
- `ADMIN_API_TOKEN` 使用 Bearer 鉴权；`operation_command` 同时保存幂等键、请求指纹、操作者、原因、request_id、客户端地址和结果。
- 前端已接入真实数据、数据发布页、人工操作对话框、管理 Token 输入、加载、空状态和错误反馈。
- 已通过 50 个默认服务端测试、2 个真实 PostgreSQL 运维集成测试、Alembic 一致性检查，以及前端 lint、类型检查、5 个测试和生产构建。
