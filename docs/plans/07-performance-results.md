# 目标规模性能验收

状态：已通过

## 验收规模

本轮使用隔离 PostgreSQL 实例生成完整宽度的股票日线和运行数据，不调用 Tushare，不使用 Docker。

| 数据 | 数量 |
| --- | ---: |
| 股票日线 `stock_daily` | 10,010,000 行，5,500 只股票、1,820 个交易日 |
| 采集批次 | 20,000 |
| 采集任务 | 500,000 |
| 加工任务 | 100,000 |
| 加工依赖 | 300,000 |
| Tushare 请求记录 | 1,000,000 |
| 数据发布记录 | 50,000 |

测试数据库开启 PostgreSQL 默认持久化，不关闭 `fsync`。正式验收命令为：

```bash
make performance
```

脚本会创建专用临时数据库 `stock_data_sync_perf`，执行最新迁移，生成目标规模数据，运行基准后停止并删除临时实例。结果保存到 `dist/performance/target.json`。任何其他数据库名称都会被测试程序拒绝。

## 验收规则

| 路径 | P95 上限 |
| --- | ---: |
| 单交易日 5,500 行 COPY 暂存、校验和分区替换发布 | 3,000 ms |
| 按交易日和按股票查询日线 | 250 ms |
| 采集与加工队列领取 | 100 ms |
| 每轮关闭完成批次 | 250 ms |
| 每轮增量规划 100 个真实依赖批次 | 1,000 ms |
| 20,000 个已规划历史批次空转检查 | 100 ms |
| 各管理后台查询 | 1,000 ms |

交易日查询还必须通过执行计划验证只扫描目标月份分区。数据生成后的实际行数必须与目标规模完全一致；任一延迟、分区或行数条件失败，命令返回非零状态。
