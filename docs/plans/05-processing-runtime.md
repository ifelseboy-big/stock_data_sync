# 加工与发布链路

状态：已完成

## 目标

批次关闭后统一解析必要依赖，并通过统一入口受控并发完成清洗、校验、正式写入和数据发布；同一输出数据集保持串行。

## 实现任务

1. 根据 `DatasetSpec` 幂等生成加工任务和确定性输出版本。
2. 持久化 RAW_ASSET 与 DATASET_RELEASE 依赖解析结果。
3. 实现 WAITING_DEPENDENCY、QUEUED、RUNNING、RETRY_WAIT、BLOCKED 和终态迁移。
4. 使用数据库 advisory lock、运行数上限、输出数据集互斥和稳定排序领取加工任务。
5. 建立处理器协议和临时表批量写入框架。
6. 实现 UPSERT_KEY、REPLACE_DATE、REPLACE_ENTITY、MASTER_MERGE 和 PATCH_COLUMNS。
7. 将质量门禁、正式表写入和 `dataset_release` 更新放入同一事务。
8. 实现受影响范围重算，禁止加工重试再次请求 Tushare。

## 验收条件

- 任意时刻 RUNNING 任务数不超过配置上限，同一输出数据集最多一个。
- 所有必要依赖 READY 前不能进入队列。
- 加工失败不产生半成品发布记录。
- 补采或规则升级只创建受影响范围的新加工版本。

## 实施记录

- 已实现确定性加工版本、RAW_ASSET/DATASET_RELEASE 依赖、全局 advisory lock、受控并发稳定队列、持久化重试和启动恢复。
- 已实现 PostgreSQL 会话临时表发布框架及五种写入策略；正式写入、数据库质量门禁、发布记录和任务终态处于同一事务。
- 已交付交易日历、股票、上市公司、股票核心日行情、涨跌停补列、技术因子、资金流、停复牌及 ETF 主数据、ETF 日线、ETF 份额规模共 11 个处理器，对应当前启用的 14 个 Tushare 接口。
- 部分补采优先使用本批次资产，并按批次关闭时点锁定其他必要接口的最新有效资产；同范围下游自动重算，加工重试不调用 Tushare。
- 真实链路验证发布 5866 条股票、5851 条公司、5522 条核心日行情、5522 条涨跌停补列、5522 条技术因子、5195 条资金流和 7 条停复牌记录；全部生成对应 release。
- 默认测试 50 通过、4 个外部集成按环境开关跳过；PostgreSQL 依赖/串行/回滚测试、Tushare 真实采集加工测试和 Alembic 漂移检查均通过。
