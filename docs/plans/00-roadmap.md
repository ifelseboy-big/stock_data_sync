# 总体实现路线

状态：实现中

## 目标

## 实施规则

- 每个阶段单独迁移、测试和验收，不能用后续阶段掩盖当前阶段未完成项。
- 首次正式上线前以最新数据库基线为准；测试库直接重建，不实现旧结构兼容、历史数据搬迁或多版本升级分支。
- 外部请求、Parquet 读写和长计算不占用数据库事务。
- 先建立状态与幂等约束，再实现 Scheduler 领取和人工操作。
- 每完成一个阶段，更新对应计划状态、验收结果和提交编号。
- 未满足阶段验收条件时，不把页面占位、空 Repository 或手工 SQL 视为完成。
