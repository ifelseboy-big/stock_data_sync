# 管理后台设计

## 定位

管理后台是数据同步系统的运维与观测入口，不承担股票行情分析。它用于回答四类问题：系统现在是否正常、任务运行到哪里、失败或阻塞发生在哪里、是否需要人工介入。

## 功能结构

### 运行中心

- 运行概览：展示正在采集的批次、当前加工任务、加工排队与阻塞数量、任务成功率和接口健康摘要。
- 采集运行：按采集批次查看任务完成情况、自动补采结果、失败数量与耗时。
- 加工队列：展示唯一运行槽位、全局串行队列、优先级、等待重试和阻塞原因。
- 依赖观测：展示加工任务依赖的原始数据是否就绪、使用当前周期还是最新有效数据、缺失原因。

### 观测中心

- 接口监控：展示 Tushare 的每分钟额度、当前窗口用量、被限流延迟数，以及各接口的成功率和耗时分位数。
- 运行记录：查询采集与加工任务的执行实例、重试次数、耗时及错误摘要。
- 告警中心：聚合任务失败、依赖阻塞、接口异常和系统异常；确认告警属于后续写操作能力。
- 系统运维：展示后端、数据库、调度器等组件状态和版本信息；启停服务由安装目录中的运维命令负责，不在浏览器中直接执行。

## 指标口径

| 指标 | 口径 |
| --- | --- |
| 采集中批次 | 当前状态为运行中，且尚未关闭的采集批次数量 |
| 加工中任务 | 当前占用全局唯一加工槽位的任务数量，正常情况下为 0 或 1 |
| 阻塞任务 | 因必要依赖未就绪而无法进入加工队列的任务数量 |
| 今日任务成功率 | 今日已进入终态的任务实例中，成功实例数 / 终态实例数；运行中、排队中和等待重试不进入分母 |
| 今日接口成功率 | 今日已完成的 Tushare 请求中，业务判定成功的请求数 / 已完成请求数；限流等待尚未发出时不进入分母 |
| 接口 P95 耗时 | 今日已完成请求从发出到收到完整响应的耗时第 95 百分位，不包含限流器中的排队等待 |
| 任务耗时 | 任务实际开始执行至进入终态的墙钟时间；队列等待和重试退避分别记录，不混入执行耗时 |
| 批次耗时 | 首个采集任务开始至所有采集任务进入终态并关闭批次的墙钟时间 |
| 当前窗口额度 | Tushare 最近 60 秒滑动窗口内已实际发出的请求数量，额度上限固定从服务端配置读取 |

所有比率在分母为零时返回 `null`，前端显示 `--`，不显示为 0%，避免把“没有样本”误解为“全部失败”。所有时间由服务端返回带时区的 ISO 8601 字符串，前端按用户时区展示。

## 前后端边界

前端按业务页面分模块，统一从 `modules/operations/api.ts` 读取运维视图，接口类型集中在 `contracts.ts`。这些接口是只读查询模型，不直接复用任务执行域对象，避免后台展示需求反向耦合任务执行实现。

当前阶段只实现查询契约、页面和加载/空数据/异常状态。任务重跑、告警确认、优先级提升等写操作，需在权限、审计和幂等规则明确后接入。
